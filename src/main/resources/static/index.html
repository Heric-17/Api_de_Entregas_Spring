Resumo


Esta é uma API Rest criada para demonstrar conhecimento sobre o Framework Spring, padrões de projeto, boas práticas de desenvolvimento e domínio sobre a linguagem Java.


Ela consiste em uma simulação da solicitação de entregas para determinado destinatário realizado a partir do cliente. A entidade entrega possui: data da solicitação e finalização geradas dinamicamente com o padrão ISO8601 com OFFSET, status atual podendo ser Pendente, Finalizada ou Cancelada, destinatário que terá o endereço da entrega, uma lista de Ocorrências com uma descrição e o horário que foi realizada, e por fim um cliente com nome, e-mail e telefone.


A aplicação foi separada em dois pacotes: “Api” e “Model”, sendo esse um modelo para separar a implementação da API, das regras de negócio. Utilizando o padrão DTO foram separadas as entidades de sua representação, tornando a manutenção e usabilidade da Api mais simples, com classes para entrada e para apresentação dos objetos.


Por fim, para ter maior controle da aplicação foi utilizado o Flyway deixando transparente as decisões e alterações no banco de dados.


Tecnologias utilizadas


  • Java: Linguagem Core da Api;
  • Rest: Normas seguidas para construção da api;
  • Maven: Gerenciador de dependências e construção do projeto;
  • Spring Boot: Configuração do ambiente de desenvolvimento;
  • Spring Data: Persistência e validação dos dados utilizando Hibernate;
  • Lombok: Gerardor de códigos padrão do Java;
  • MySQL: Banco de Dados para armazenamento;
  • Flyway: Gerenciamento das tabelas do banco de dados;
  • ModelMapper: Biblioteca para facilitar a implementação da Representation Model;
  • MVC(Model View Controller): Padrão utilizado para organização do projeto;
  • Repository: Padrão utilizado para centralizar as classes de acesso a dados;
